# Planning Methodology

## 기획 프로세스
1. **맥락 파악**: 요구사항의 배경, 동기, 제약조건 이해
2. **현황 분석**: 코드베이스 탐색, 기존 아키텍처 파악
3. **스코프 정의**: 무엇을 하고 무엇을 하지 않을지 경계 설정
4. **설계**: 기술적 구현 방안 (데이터 흐름, API, 스키마)
5. **리스크 식별**: 예상 문제점과 대안
6. **분할**: 적절한 작업 단위로 이슈 분할
7. **우선순위**: 의존성과 가치 기반 순서 결정

## 코드베이스 분석 체크리스트
- [ ] 프로젝트 구조 (디렉토리, 주요 모듈)
- [ ] 기술 스택 (언어, 프레임워크, 도구)
- [ ] 설정 패턴 (config, env, schema)
- [ ] 테스트 패턴 (프레임워크, 구조, 커버리지)
- [ ] 기존 이슈/PR 현황
- [ ] README/CLAUDE.md 확인

## 요구사항 분석
- 기능 요구사항: 무엇이 동작해야 하는가
- 비기능 요구사항: 성능, 보안, 확장성
- 제약 조건: 시간, 기술, 호환성
- 가정: 전제 조건 명시

## 설계 원칙
- KISS: 단순한 설계를 우선
- YAGNI: 필요하지 않은 것은 만들지 않음
- SRP: 하나의 모듈은 하나의 책임
- DIP: 구체 구현이 아닌 추상에 의존
- 점진적 개선: 완벽보다 동작하는 것을 먼저
